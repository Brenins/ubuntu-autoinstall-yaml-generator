Official Documentation

Identity

Main user, hostname and password configuration. Autoinstall identity block.

Machine name on network (no spaces or special characters)
User real name (display name)
Login user (lowercase letters, no spaces)
Will be converted to SHA-512 hash in generated YAML

Localization

Language and timezone settings.

Keyboard

Keyboard layout and variant.

Ex: "intl", "nodeadkeys", "dvorak" (leave empty for default)

Network

Network configuration via Netplan.

Network interface name (ex: eth0, ens3, enp0s3)
Address with mask (ex: 192.168.1.100/24)
Separate multiple with comma
Comma-separated search domains

Storage

Disk and partitioning configuration.

Block device (ex: /dev/sda, /dev/vda, /dev/nvme0n1)

Custom Partitions

SSH

Remote access via OpenSSH.

One key per line. Supported formats: ssh-rsa, ssh-ed25519, ecdsa-sha2-*
Fetches public keys from the specified account

Additional Users

Extra users via cloud-init. Main user is in Identity.

Packages

Packages to install after the initial setup.

One package per line. Accepts valid APT package names.

Snaps

Snap packages to install.

Popular Snaps

Updates

Update policy during installation.

security: Installs only security updates. all: Installs all.

Early-Commands

Commands run before installation starts.

One command per line. These commands run in the Curtin environment before installation.

Common Commands

Late-Commands

Commands run after installation. Target at /target.

One command per line. Use curtin in-target --target=/target -- to execute on the installed system.

Common Commands

Advanced

Additional settings.

Autoinstall schema version. Currently only v1 is supported.
Equivalent to drivers: install: true
Equivalent to codecs: install: true
Equivalent to oem: install: auto
Proxy for use during installation. Block proxy.
Equivalent to shutdown: poweroff (default is reboot)
